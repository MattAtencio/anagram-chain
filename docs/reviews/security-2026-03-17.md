# Security Review: Anagram Chain

**Date:** 2026-03-17
**Reviewer:** Automated (Claude Code)
**Scope:** Full codebase
**Overall Rating:** LOW RISK

---

## Executive Summary

Anagram Chain is a client-side-only PWA word puzzle game with no backend, no API routes, no authentication, and no user-generated content that is rendered as HTML. The attack surface is minimal. No critical or high severity issues were found. A handful of low and informational findings are documented below.

## Findings

### [LOW] L-1: Vercel Project Credentials in Repository

**File:** `.vercel/project.json`
**Description:** Contains `projectId` and `orgId` for the Vercel deployment. While these are not secret keys (they cannot be used to deploy or modify the project without authenticated CLI access), they expose internal project identifiers.
**Recommendation:** Add `.vercel` to `.gitignore`. Note: `.vercel` is already listed in `.gitignore` (line 47), so this is already mitigated if the file was added before the gitignore rule. Verify the file is not tracked by running `git ls-files .vercel/`.

---

### [LOW] L-2: localStorage Data Tampering

**File:** `components/AnagramGame.jsx` (lines 20-37)
**Description:** Game state (stats, streaks, completion status, times) is stored in localStorage and loaded without validation beyond JSON parsing. A user can manually edit localStorage to fake streaks, times, or completion status.
**Impact:** Minimal -- this is a single-player casual game with no leaderboard or competitive element. The "share results" feature copies text to clipboard, so faked results would only affect social sharing credibility.
**Recommendation:** Acceptable risk for the game's scope. If competitive features (leaderboards) are ever added, move to server-validated state.

---

### [LOW] L-3: All Puzzle Answers Are Client-Side

**Files:** `data/chains.js`, `data/bonusWords.js`
**Description:** All 90 chain definitions with every answer word are shipped in the client bundle. Any user can view them in browser DevTools or the page source.
**Impact:** Players can cheat by inspecting the source. This is inherent to a no-backend architecture.
**Recommendation:** Acceptable for the current design. If answer secrecy becomes important, consider a lightweight API that returns only the current day's scrambled letters and validates answers server-side.

---

### [INFO] I-1: No Content Security Policy (CSP) Headers

**File:** `app/layout.js`, `next.config.mjs`
**Description:** No CSP headers are configured. While the app has no dynamic content injection vectors, a CSP would provide defense-in-depth against future XSS if features are added.
**Recommendation:** Add security headers in `next.config.mjs`:
```js
const nextConfig = {
  reactStrictMode: true,
  headers: async () => [{
    source: "/(.*)",
    headers: [
      { key: "X-Content-Type-Options", value: "nosniff" },
      { key: "X-Frame-Options", value: "DENY" },
      { key: "Referrer-Policy", value: "strict-origin-when-cross-origin" },
    ],
  }],
};
```

### [INFO] I-2: Service Worker Uses `self.skipWaiting()`

**File:** `public/sw.js`
**Description:** The generated service worker calls `self.skipWaiting()` which immediately activates new service workers. This is standard PWA behavior but means cached content updates immediately without user confirmation.
**Impact:** No security impact. This is the expected behavior for this type of app.

---

### [INFO] I-3: No Input Sanitization on Share Text

**File:** `components/AnagramGame.jsx` (lines 474-498)
**Description:** The `shareResults` function constructs share text from game state and passes it to `navigator.share()` or `navigator.clipboard.writeText()`. The data originates entirely from hardcoded game constants and numeric counters, not from user-typed free text.
**Impact:** No XSS or injection risk -- the text is composed of controlled strings and numbers.

---

### [INFO] I-4: `alert()` Used for Clipboard Feedback

**File:** `components/AnagramGame.jsx` (line 496)
**Description:** Uses `alert("Copied to clipboard!")` as a fallback notification. This is functional but blocks the UI thread.
**Recommendation:** Consider replacing with a toast/snackbar notification for better UX (not a security concern).

---

### [INFO] I-5: PWA Icon Files Are Placeholder Size (70 bytes)

**Files:** `public/icon-192.png`, `public/icon-512.png`
**Description:** Both icon files are only 70 bytes, which suggests they are placeholders or corrupted. This is not a security issue but may affect PWA installability and trust signals.

## XSS / Injection Analysis

- **No `dangerouslySetInnerHTML`** usage anywhere in the codebase.
- **No `eval()`, `Function()`, or `innerHTML`** assignments.
- **No user free-text input** -- all interaction is through single-character letter taps or keyboard keys filtered to `[A-Z]`, `Enter`, and `Backspace`.
- **React's JSX escaping** handles all rendering. All displayed text is from hardcoded constants or numeric state.

**Verdict:** No XSS vectors identified.
